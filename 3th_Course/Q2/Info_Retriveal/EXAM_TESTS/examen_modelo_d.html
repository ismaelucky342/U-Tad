A
Parquet es legible directamente por humanos, lo que facilita la depuración.
B

C
CSV comprime mejor que Parquet en disco.

¿Qué es el TF-IDF y qué problema resuelve respecto al simple conteo de palabras (TF)?
A
TF-IDF convierte el texto en embeddings semánticos; TF no puede capturar significado.
B
TF-IDF penaliza palabras muy frecuentes en todo el corpus (baja discriminación), potenciando las palabras específicas de cada documento.
C
TF-IDF elimina automáticamente las stopwords antes de contar.
D
TF-IDF solo funciona con textos largos; TF es más adecuado para oraciones cortas.
Correcto: B. TF mide la frecuencia de una palabra en un documento. El problema es que palabras como "el" o "de" son frecuentes en todos los documentos. IDF penaliza esas palabras ubicuas y premia las que aparecen en pocos documentos, haciéndolas más discriminativas.

¿Qué hace model.eval() en PyTorch antes de la inferencia?
model.eval()
with torch.no_grad():
    outputs = model(**inputs)
A
Congela los pesos del modelo para que no se actualicen durante la inferencia.
B
Desactiva capas como Dropout y BatchNorm que se comportan diferente en entrenamiento vs inferencia.
C
Mueve el modelo de GPU a CPU para la inferencia.
D
Evalúa el modelo sobre el dataset de validación automáticamente.
Correcto: B. model.eval() cambia el comportamiento de capas como Dropout (que se desactiva) y BatchNorm (que usa estadísticas del batch de entrenamiento). torch.no_grad() desactiva el cálculo de gradientes para ahorrar memoria. Ambos son necesarios para inferencia correcta.

¿Qué representa el parámetro attention_mask que genera el tokenizador de Hugging Face?
A
Los pesos de atención aprendidos por el modelo durante el preentrenamiento.
B
Una máscara binaria que indica qué tokens son reales (1) y cuáles son padding (0).
C
La posición de los tokens especiales [CLS] y [SEP] en la secuencia.
D
Una máscara que oculta tokens futuros para el entrenamiento causal (como GPT).
Correcto: B. Cuando se hace padding para igualar longitudes en un batch, attention_mask indica qué posiciones contienen tokens reales (1) y cuáles son relleno (0). El modelo ignora los tokens de padding en el cálculo de atención gracias a esta máscara.

¿Qué es el fine-tuning de un modelo preentrenado como BERT?
A
Entrenar BERT desde cero con un dataset específico de la tarea.
B
Continuar entrenando un modelo preentrenado con datos etiquetados de la tarea destino, ajustando sus pesos.
C
Añadir capas de atención adicionales sin modificar los pesos originales de BERT.
D
Comprimir BERT para que ocupe menos memoria sin modificar su comportamiento.
Correcto: B. El fine-tuning (ajuste fino) parte de los pesos preentrenados de BERT y continúa el entrenamiento con datos etiquetados de la tarea específica. Los pesos del modelo se actualizan levemente. Permite aprovechar el conocimiento preentrenado con relativamente pocos datos.

¿Qué función de activación es más adecuada en la capa de salida de una red neuronal para clasificación binaria?
A
ReLU
B
Sigmoid
C
Softmax
D
Tanh
Correcto: B. Sigmoid mapea cualquier valor a (0, 1), interpretable como probabilidad de clase positiva. Para clasificación multiclase se usa Softmax. ReLU y Tanh se usan en capas ocultas, no en la capa de salida de clasificación. Dense(1, activation='sigmoid') es el estándar para clasificación binaria.

¿Qué ocurre si el parámetro max_length del tokenizador es menor que la longitud real del texto de entrada?
inputs = tokenizer("Texto muy largo...", max_length=64, truncation=True)
A
El tokenizador lanza un error porque el texto no cabe.
B
El texto se trunca a 64 tokens, descartando los tokens finales que excedan ese límite.
C
El tokenizador ignora max_length si es menor que el texto real.
D
El texto se divide automáticamente en múltiples chunks de 64 tokens.
Correcto: B. Con truncation=True, el tokenizador trunca el texto a max_length tokens, descartando el exceso por la derecha (por defecto). Sin truncation=True, ignoraría max_length. Para dividir en chunks se necesita lógica adicional con stride.

¿Para qué sirve el contexto torch.no_grad()?
A
Evita que el modelo actualice sus pesos durante el entrenamiento.
B
Desactiva el cálculo y almacenamiento de gradientes, reduciendo el uso de memoria y acelerando la inferencia.
C
Congela capas específicas del modelo para el fine-tuning.
D
Detiene el entrenamiento si la loss no converge.
Correcto: B. Durante inferencia o evaluación, no necesitamos calcular gradientes. torch.no_grad() desactiva el autograd engine: no se construye el grafo computacional ni se almacenan gradientes intermedios. Esto reduce el uso de memoria GPU hasta 2x y acelera el cómputo.

¿Qué es el mecanismo de self-attention en los Transformers?
A
Un mecanismo recursivo donde cada capa atiende a la capa anterior de la red.
B
Un mecanismo que permite a cada token de la secuencia "atender" a todos los demás tokens para calcular su representación contextual.
C
Una capa que reduce la dimensionalidad de los embeddings mediante matrices de proyección.
D
Un mecanismo donde solo el token [CLS] atiende al resto de la secuencia.
Correcto: B. En self-attention, cada token calcula puntuaciones de atención con todos los demás tokens de la misma secuencia (incluyendo él mismo). Esto permite capturar dependencias a larga distancia, algo que las RNNs tenían dificultad para hacer. El resultado es una representación de cada token en su contexto completo.

¿Qué diferencia hay entre BertForMaskedLM y BertForSequenceClassification en Hugging Face?
A
Son arquitecturas completamente diferentes sin ninguna parte en común.
B
Ambas usan el mismo encoder BERT, pero añaden cabezas de predicción distintas según la tarea.
C
BertForMaskedLM es más grande y potente para todas las tareas.
D
BertForSequenceClassification no puede cargarse con pesos preentrenados.
Correcto: B. El encoder BERT es compartido. Lo que cambia es la "cabeza" (head): BertForMaskedLM añade una capa de predicción sobre el vocabulario para rellenar máscaras [MASK]. BertForSequenceClassification añade un clasificador lineal sobre el token [CLS] para clasificar la secuencia completa.
